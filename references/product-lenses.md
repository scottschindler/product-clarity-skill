# ProductClarity Product Lenses

Use these lenses depending on the user's goal.

## Founder Lens

Prioritize:

- wedge
- target user
- painful problem
- monetization
- launch strategy
- riskiest assumption
- speed to market

Use when the user is exploring a startup, business, SaaS, consumer product, or monetizable tool.

## Designer Lens

Prioritize:

- user flow
- screens
- first magical moment
- empty states
- interaction design
- emotional experience
- clarity of output

Use when the user asks about UX, UI, onboarding, or how the product should feel.

## Engineer Lens

Prioritize:

- data model
- system architecture
- API behavior
- build phases
- implementation scope
- Cursor-ready prompt

Use when the user wants to build the product quickly.

## Growth Lens

Prioritize:

- target audience
- launch channels
- viral loop
- SEO/social/community angles
- shareable output
- referral mechanics

Use when the user asks how this spreads or makes money.

## Investor Lens

Prioritize:

- market size
- urgency
- differentiation
- defensibility
- business model
- wedge expansion
- why now

Use when the user wants to evaluate whether the idea is worth pursuing.

## Operator Lens

Prioritize:

- workflow
- repeat usage
- reliability
- edge cases
- integrations
- internal processes
- measurable productivity gain

Use when the product is for teams, agencies, businesses, or internal operations.
