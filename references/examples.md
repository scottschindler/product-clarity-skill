# ProductClarity Examples

## Example 1: Vague idea

User:

I want to build an app that helps people improve their focus. Maybe it audits their attention span and gives recommendations.

Expected behavior:

Ask clarifying questions if the user wants to think through it. If they ask for a spec, assume the first user is ambitious knowledge workers and produce a spec.

Good response:

- Defines target user clearly
- Narrows MVP to one focus diagnostic
- Includes non-goals
- Includes viral result page
- Includes pricing
- Includes Cursor build prompt

Bad response:

- Lists 20 possible features
- Suggests generic meditation reminders
- Does not define the first user
- Does not define a magic moment

## Example 2: Scattered founder notes

User:

I want something like a GTM engineer as a service. You plug in your app and it automates Reddit, X, SEO, Discord, posting, etc.

Expected behavior:

Turn into a product spec for a GTM automation command center.

Good response:

- Defines first audience as solo founders or early-stage SaaS teams
- Narrows wedge to one acquisition channel first
- Separates agent actions from recommendations
- Includes approval workflows
- Includes launch plan

Bad response:

- Says “automate all marketing”
- Ignores risk of spam
- Ignores integrations
- Skips guardrails

## Example 3: Feature idea

User:

Add a daily challenge feature to my writing app where everyone gets the same image and writes a description.

Expected behavior:

Produce feature spec, not full company spec.

Good response:

- Defines user flow
- Defines daily image assignment
- Defines submission, scoring, comparison, sharing
- Defines data model
- Defines edge cases

Bad response:

- Turns it into a broad social network
- Adds unrelated writing features
